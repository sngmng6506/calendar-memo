# 업무 · 일정 관리

여러 목표·회사 업무, 전체 업무량, 하루 컨디션을 같은 날짜 축에서 보는 웹앱입니다.
로그인 없이 브라우저 `localStorage`로 동작하며, 구글 로그인 시 Railway Postgres에 자동 저장되어 다른 PC에서도 이어서 볼 수 있습니다.

## 실행

**오프라인(로컬 전용):** `index.html`을 브라우저로 열면 됩니다(동기화 없이 localStorage).

```powershell
start .\index.html
```

**서버 모드(동기화 포함):**

```powershell
npm install
# .env 작성(아래 환경변수 참고) 후
npm start
# http://localhost:3000
```

## 클라우드 동기화 (Railway + 구글 로그인)

구조: Express 서버(`server.js`)가 정적 프론트도 함께 서빙 + `/api/*` 제공. 보드는 사용자별 1행 JSONB로 Postgres `boards` 테이블에 저장. 로그인 시 **서버 우선, 서버가 비어있으면 현재 로컬 보드를 올려 병합**.

설정 순서:

1. **Google Cloud Console** → OAuth 2.0 클라이언트 ID(웹) 생성
   - 승인된 자바스크립트 출처: 배포 도메인(예: `https://<앱>.up.railway.app`)과 로컬 테스트용 `http://localhost:3000`
2. **Railway** → 새 프로젝트에 이 저장소 연결 + **PostgreSQL** 추가(`DATABASE_URL` 자동 주입)
3. Railway 서비스 **Variables**:
   - `GOOGLE_CLIENT_ID` = 1번에서 만든 클라이언트 ID
   - `JWT_SECRET` = 길고 무작위한 문자열
   - `NODE_ENV` = `production`
4. 배포 후 접속 → 우상단 구글 로그인 → 자동 저장/불러오기

환경변수 예시는 `.env.example` 참고. `GOOGLE_CLIENT_ID`나 `DATABASE_URL`이 없으면 동기화가 비활성화되고 자동으로 localStorage 전용으로 동작합니다.

## 현재 기능

- `Timeline`에서 목표 추가
- 목표 추가 시 `회사 프로젝트`로 표시 가능
- 목표 이름, 시작일, 종료일 등록
- 기본 목표 순서는 종료일 임박순
- 목표 row 드래그로 수동 순서 변경
- row 헤더에서 이름, 기간, 소요 시간 수정
- 컨디션 행 바로 밑에 상시 `회사 업무` 고정 행 (순서변경·삭제 불가, 소요 시간만 수정)
- 회사 업무는 평일이면 기록 없이도 업무량에 기본 반영(주말·공휴일은 기록한 날만 반영)
- 회사 프로젝트는 `회사 업무` 하위에 접기/펼치기 가능한 row로 표시
- 회사 프로젝트는 Timeline과 목표별 흐름에서 따로 관리하되, 업무량은 큰 `회사 업무`에 포함되어 별도 소요 시간을 갖지 않음
- 정기 일정 등록 시 등록일 기준 지정 기간(최소 1개월) 동안 매주 선택한 여러 요일에 반복 표시
- 날짜 셀 클릭/더블클릭으로 마일스톤, 트러블, 추가 일, 메모, 진척률 증감 기록
- 기록이 있는 날짜 셀을 같은 목표 row 안에서 드래그해 다른 날짜로 이동
- 기간 밖 날짜 셀에 기록하면 목표 기간 자동 확장
- 진척률 증감은 noUiSlider 기반 0 중심형 슬라이더로 입력
- 진척률 양수는 파랑, 음수는 빨강, 0은 포인트로 표시
- `내 컨디션` row에서 날짜별 컨디션 기록
- 컨디션은 0-120 슬라이더로 설정
- 컨디션 이모지 기준점은 40/60/80/100에 표시
- 컨디션 슬라이더는 화면상 `0 / 40 / 60 / 80 / 100 / 120` 구간으로 환산
- 이모지 기준점 근처에서는 자석처럼 스냅
- 컨디션 미기록일은 실제 기록값을 저장하지 않고 화면에서만 추정 표시
- 기록 사이의 컨디션은 가까운 기록에 exponential 가중치를 둔 smoothing으로 추정
- 마지막 기록 이후 컨디션은 최근 추세와 완만한 sine 감쇠를 반영한 점선 예측으로 표시
- 전체 흐름 업무량 wave는 해당 날짜에 기록/정기 일정이 있는 목표들의 소요 시간을 12시간 기준으로 환산해 표시
- 전체 흐름에는 업무량 wave와 컨디션 선을 함께 표시
- 전체 흐름의 왼쪽 여백에는 이전 날짜 흐름을 점선으로 미리 표시
- 목표별 흐름은 각 목표의 마일스톤 날짜만 누적진척률 지점에 다이아몬드로 표시하고, 같은 목표의 마일스톤끼리 점선으로 연결
- 그래프 상단 범례는 모드별로 표시 (전체: 업무량·컨디션 / 목표별: 마일스톤)
- 그래프 hover 시 날짜별 업무량, 컨디션, 상태, 목표별 수치 표시
- 그래프 또는 타임라인에서 날짜를 hover하면 양쪽 날짜가 함께 강조 표시
- 그래프에 오늘 날짜 marker 표시
- 그래프 배경에 주말(토/일)·공휴일 칼럼을 옅은 붉은색으로 표시
- 그래프와 타임라인이 같은 날짜 창을 공유
- 그래프/날짜 영역 드래그 또는 이전/다음 버튼으로 날짜 이동
- 토요일은 파랑, 일요일과 공휴일은 빨강으로 표시
- 대한민국 공휴일은 `holidays/<연도>.js`에서 연도별로 로드 (`file://`에서도 동작, 파일 없으면 양력 고정 공휴일로 폴백)
- 브라우저 `localStorage` 저장
- 샘플 데이터 초기화
- 데스크톱/모바일 반응형 화면

## 데이터 저장 방식

데이터는 현재 브라우저의 `localStorage`에 저장됩니다.

저장 키:

```text
schedule-wave-board
```

저장 구조:

- `tasks`: 대목표/업무 목록, 기간, 소요 시간, 반복 설정, 날짜별 기록
- `energy`: 날짜별 에너지 값
- `taskOrderMode`: 종료일 임박순 또는 수동 정렬 상태

업무량은 소요 시간을 `소요 시간 / 12시간 * 100`으로 환산해 계산합니다. 캘린더에 업무가 있는 날짜에만 해당 목표의 소요 시간이 반영되며, 하루 업무량은 100%를 초과할 수 있습니다.

공휴일 데이터는 `holidays/<연도>.js` 파일로 분리되어 있습니다. 화면에 보이는 연도만 `<script>`로 동적 주입해 불러오고 캐시합니다. `<script>` 방식이라 `fetch`와 달리 `file://`(파일 더블클릭 실행)에서도 차단 없이 동작합니다.

새 연도는 같은 형식의 파일을 추가하면 됩니다(HTML 수정 불필요).

```js
// holidays/2027.js
registerHolidays(2027, {
  "2027-01-01": "신정",
  // ...
});
```

음력 공휴일·대체공휴일·임시공휴일은 규칙만으로 정확히 산출되지 않으므로, 각 연도 파일은 공공데이터포털 KASI 특일정보 API 결과로 미리 생성해 두는 방식을 권장합니다. 해당 연도 파일이 없으면 자동으로 양력 고정 공휴일(신정·삼일절·어린이날·현충일·광복절·개천절·한글날·성탄절)로 폴백합니다.
