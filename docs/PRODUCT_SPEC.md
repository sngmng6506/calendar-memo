# Product Specification

## 1. Problem

업무를 캘린더, 할 일 앱, 메모, 업무보고 문서에 각각 다시 입력하면 기록 비용이 커진다. 특히 업무보고를 위해 한 주의 기록을 다시 찾고 정리하는 과정이 반복된다.

## 2. Product Goal

날짜 셀에 남긴 체크리스트 자체를 업무 원장으로 사용하고, 선택 기간의 기록을 AI가 제출 가능한 보고서로 변환한다.

## 3. Target User

- 개인 업무 기록을 빠르게 남기고 싶은 사무·연구·개발 직군
- 별도의 프로젝트 관리 도구까지는 필요하지 않은 사용자
- 주간 또는 월간 업무보고를 반복 작성하는 사용자

## 4. MVP Functional Requirements

### FR-01 월간 달력

- 월요일 시작, 7열, 고정 6주인 42개 날짜 셀을 표시한다.
- 이전·다음 달 날짜는 같은 그리드에 흐리게 표시한다.
- 이전 달, 다음 달, 오늘 이동을 제공한다.

### FR-02 날짜별 업무 입력

- 각 날짜 셀에는 기존 업무와 항상 하나 이상의 빈 입력 행이 있다.
- 빈 행에 내용을 입력하고 `Enter`를 누르면 저장한다.
- 저장 직후 같은 날짜의 다음 입력 행에 포커스를 둔다.
- 업무는 한 줄 텍스트다.

### FR-03 완료 체크

- 모든 저장된 업무 왼쪽에 체크박스를 표시한다.
- 체크 변경은 즉시 저장한다.
- 완료 업무는 흐린 글자로 표시한다.

### FR-04 편집과 삭제

- 업무 텍스트를 직접 수정할 수 있다.
- 포커스를 잃거나 `Enter`를 누르면 저장한다.
- 기존 업무를 빈 문자열로 만들면 삭제한다.

### FR-05 미완료 이동

- 어제의 미완료 업무만 오늘의 마지막 순서로 이동한다.
- 완료 업무는 어제 날짜에 남긴다.

### FR-06 보고서

- 일간, 주간, 월간 기간을 지원한다.
- 로컬 규칙 기반 미리보기는 오프라인으로 동작한다.
- 사용자가 명시적으로 버튼을 누를 때만 업무 기록을 LLM API에 보낸다.
- 생성 결과를 편집하고 클립보드에 복사할 수 있다.

### FR-07 로컬 저장

- SQLite를 사용한다.
- 계정과 서버 없이 동작한다.
- 앱 재시작 후 기록과 체크 상태를 복원한다.

## 5. Non-functional Requirements

- 외부 Python 패키지 없이 실행 가능해야 한다.
- 빈 상태의 앱은 2초 이내에 첫 화면을 표시하는 것을 목표로 한다.
- 네트워크 연결 없이 업무 기록 기능 전체가 동작해야 한다.
- 데이터베이스 파일을 복사해 백업할 수 있어야 한다.
- API 키를 영구 저장하지 않는다.

## 6. Out of Scope

로그인, 클라우드 동기화, 협업, 태그, 프로젝트, 우선순위, 반복 일정, 알림, 모바일 앱, 외부 캘린더 연동, 채팅형 AI는 MVP 범위 밖이다.

## 7. Acceptance Scenario

1. 사용자가 2026년 7월 16일 셀에 `코터 비전 회의`를 입력한다.
2. `Enter`를 누르면 업무가 저장되고 바로 아래 빈 입력 행으로 이동한다.
3. `Y-profile 데이터 정리`를 입력한다.
4. 첫 업무의 체크박스를 선택한다.
5. 주간 보고서를 열어 로컬 미리보기를 누른다.
6. 완료 섹션에는 `코터 비전 회의`, 남은 업무에는 `Y-profile 데이터 정리`가 표시된다.
7. LLM 생성 버튼을 누르면 동일 기록만 근거로 문장형 보고서가 생성된다.
