# Product Specification

## 1. Problem

업무를 캘린더, 할 일 앱, 메모, 업무보고 문서에 반복 입력하면 기록 비용이 커진다. 별도 앱 창을 찾아 열어야 하는 마찰도 매일 누적된다.

## 2. Product Goal

바탕화면 달력의 날짜 셀에 남긴 체크리스트를 업무 원장으로 사용하고, 선택 기간의 기록을 제출 가능한 보고서로 변환한다.

## 3. Target User

- 개인 업무를 빠르게 남기는 사무·연구·개발 직군
- 프로젝트 관리 도구까지는 필요하지 않은 사용자
- 주간 또는 월간 업무보고를 반복 작성하는 사용자
- Windows 바탕화면에서 일정과 할 일을 항상 확인하고 싶은 사용자

## 4. MVP Functional Requirements

### FR-01 월간 달력

- 월요일 시작, 7열, 고정 6주인 42개 날짜를 표시한다.
- 이전·다음 달 날짜는 흐리게 표시한다.
- 이전 달, 다음 달, 오늘 이동은 창의 가로 중앙에 둔다.
- 날짜 셀은 테두리나 카드 배경 없이 정렬과 여백으로 구분한다.

### FR-02 날짜별 업무 입력

- 빈 입력 행은 평소 배경과 같아 입력 상자처럼 보이지 않는다.
- 날짜 셀의 빈 영역을 클릭하면 해당 날짜 입력 행에 포커스를 둔다.
- `Enter`는 저장 후 다음 빈 행에 포커스를 둔다.
- 업무는 한 줄 텍스트다.

### FR-03 완료 체크

- 저장된 업무 왼쪽에 평면형 `□ / ✓`를 표시한다.
- 체크 변경은 즉시 저장한다.
- 완료 업무는 흐린 글자로 표시한다.

### FR-04 편집과 삭제

- 업무 텍스트를 직접 수정한다.
- 포커스를 잃거나 `Enter`를 누르면 저장한다.
- 기존 업무를 빈 문자열로 만들면 삭제한다.

### FR-05 미완료 이동

- 어제의 미완료 업무만 오늘 마지막 순서로 이동한다.
- 완료 업무는 원래 날짜에 남긴다.

### FR-06 보고서

- 일간, 주간, 월간 기간을 지원한다.
- 로컬 미리보기는 오프라인으로 동작한다.
- 사용자가 명시적으로 누를 때만 기록을 LLM API에 보낸다.
- 결과를 편집하고 클립보드에 복사할 수 있다.

### FR-07 로컬 저장

- SQLite를 사용한다.
- 계정과 서버 없이 동작한다.
- 앱 재시작 후 기록과 체크 상태를 복원한다.

### FR-08 Windows 바탕화면 통합

- Windows 첫 실행은 바탕화면 모드를 기본값으로 한다.
- Explorer의 `SHELLDLL_DefView`가 있는 창을 기준으로 뒤쪽 `WorkerW`를 찾는다.
- WorkerW가 없고 Progman이 아이콘 호스트이면 Progman으로 폴백한다.
- 연결된 모니터 목록을 읽고 사용자가 선택한 모니터 한 대의 좌표와 해상도에만 Daymark를 배치한다.
- WorkerW가 가상 데스크톱 전체를 덮더라도 선택 모니터 좌표를 부모 기준 상대 좌표로 변환한다.
- 바탕화면 연결과 유지 시 `WS_EX_LAYERED` 및 alpha 값을 다시 적용한다.
- Explorer 재시작으로 부모 핸들이 사라지면 주기적으로 재탐색한다.
- 연결 실패 시 원래 부모와 창 스타일을 복원하고 일반 창으로 동작한다.
- 우측 버튼, `Ctrl+Shift+D`, `--window`, `--desktop`으로 모드를 전환한다.
- 설정에서 표시 모니터와 투명도를 변경하고 즉시 적용한다.

## 5. Non-functional Requirements

- 외부 Python 패키지 없이 실행 가능해야 한다.
- 네트워크 없이 기록과 체크 기능이 모두 동작해야 한다.
- API 키를 영구 저장하지 않는다.
- 메인 창은 기본 0.86 반투명이며 창/바탕화면 모드에서 같은 설정값을 사용한다.
- 바탕화면 통합 실패가 데이터베이스 트랜잭션에 영향을 주지 않는다.
- WorkerW 코드와 업무 도메인 코드를 분리한다.

## 6. Out of Scope

로그인, 클라우드 동기화, 협업, 태그, 프로젝트, 우선순위, 반복 일정, 알림, 모바일 앱, 외부 캘린더 연동, 채팅형 AI는 MVP 범위 밖이다.

## 7. Acceptance Scenario

1. Windows에서 앱을 실행한다.
2. Daymark가 WorkerW 또는 Progman에 연결되어 선택한 모니터 한 대에만 표시된다.
3. 2026년 7월 16일에 `코터 비전 회의`를 입력하고 `Enter`를 누른다.
4. 다음 행에 `Y-profile 데이터 정리`를 입력한다.
5. 첫 업무를 완료 체크한다.
6. 주간 로컬 미리보기에서 완료와 미완료가 구분된다.
7. `창 모드` 또는 `Ctrl+Shift+D`로 일반 창으로 복귀한다.
8. Explorer 레이어 탐색 실패 시에도 기록은 그대로 남고 일반 창이 열린다.
