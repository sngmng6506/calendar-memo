# UX Specification

## Visual Direction

Daymark는 일반적인 일정 관리 앱의 카드 UI가 아니라, **바탕화면 위에 직접 놓인 텍스트 달력**처럼 보여야 한다.

- 기본 창은 어두운 반투명 표면이다.
- 월 이동과 현재 월 제목은 창의 정확한 가로 중앙에 둔다.
- 바탕화면 아이콘이 주로 놓이는 왼쪽에는 핵심 조작 버튼을 배치하지 않는다.
- 날짜 셀은 테두리, 흰 배경, 그림자, 둥근 카드로 구분하지 않는다.
- 날짜 간 구분은 7열 정렬과 여백만으로 만든다.
- 빈 날짜에는 입력 상자나 체크박스가 보이지 않는다.
- 셀을 클릭하면 보이지 않던 빈 입력 행에 커서가 생기고 바로 입력할 수 있다.
- 저장된 업무만 `□` 또는 `✓`와 함께 표시한다.

```text
                            ‹  2026년 7월  ›  오늘         미완료 이동  AI 요약  설정

        월              화              수              목              금              토              일

        13              14              15              16              17              18              19
                                                        □ 코터 비전 회의
                                                        ✓ Y-profile 데이터 정리
                                                        □ 특허 문구 수정
```

## Transparency

- 기본 불투명도는 `0.86`이다.
- 운영체제 창 합성을 지원하지 않으면 불투명 배경으로 폴백한다.
- 환경변수 `DAYMARK_OPACITY`로 `0.55`부터 `1.0`까지 조절할 수 있다.
- 반투명은 시각적 효과일 뿐이며 데이터 저장이나 입력 동작에 영향을 주지 않는다.

## Keyboard Rules

| 상황 | 키 | 결과 |
|---|---|---|
| 날짜 셀 클릭 | 마우스 | 해당 날짜 선택, 빈 입력 행에 포커스 |
| 빈 행에 텍스트 입력 | `Enter` | 업무 저장, 바로 다음 빈 행 생성 및 포커스 |
| 기존 업무 수정 | `Enter` | 수정 저장, 바로 아래 새 입력 행 생성 및 포커스 |
| 기존 업무가 빈 상태 | `Backspace` | 업무 삭제 |
| 기존 업무 수정 후 이동 | 포커스 아웃 | 수정 저장 |

## Visual States

- 오늘: 날짜 숫자를 파란 강조색과 굵은 글자로 표시
- 현재 월 밖 날짜: 날짜 숫자를 흐리게 표시
- 미완료 업무: `□`와 기본 글자
- 완료 업무: `✓`와 흐린 글자
- 빈 입력 행: 평소에는 배경과 동일해 보이며 포커스 시에만 약한 배경 변화
- 날짜 셀: 항상 무테두리, 무그림자, 동일 배경

## Report and Settings Dialogs

- 메인 달력과 같은 어두운 평면 색상을 사용한다.
- 입력 영역은 흰 카드가 아니라 한 단계 밝은 어두운 면으로만 구분한다.
- 보고서 유형 선택: daily / weekly / monthly
- `로컬 미리보기`: 네트워크 없이 완료·미완료 목록 생성
- `LLM으로 생성`: API 키가 있을 때 문장형 보고서 생성
- 본문은 편집 가능하며 `복사`로 전체 결과를 복사한다.

## Error UX

- API 키 없음: 설정 방법을 안내하고 요청을 보내지 않는다.
- LLM 오류: 상태 표시 후 사용자에게 오류 요약을 보여준다.
- DB 오류: 앱 시작을 중단하고 원인을 숨기지 않는다.
- 네트워크 오류가 업무 입력과 체크 기능을 막아서는 안 된다.
