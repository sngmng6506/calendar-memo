# UX Specification

## Visual Direction

Daymark는 카드형 일정 관리 앱이 아니라 **바탕화면 위에 직접 놓인 텍스트 달력**처럼 보여야 한다.

- 어두운 반투명 표면
- 월 이동과 제목은 정확한 가로 중앙
- 왼쪽 바탕화면 아이콘 영역에 핵심 조작 버튼을 두지 않음
- 날짜 셀의 테두리, 흰 배경, 그림자, 둥근 카드 금지
- 7열 정렬과 여백만으로 날짜 구분
- 빈 날짜에는 입력 상자와 체크박스를 표시하지 않음
- 저장된 업무만 원형 체크 컨트롤 표시

## Windows Desktop Mode

- Windows 기본 시작 모드는 바탕화면 모드다.
- 창 테두리와 제목 표시줄을 제거하고 선택한 모니터 한 대의 영역만 채운다.
- 상단 우측에는 `AI 요약`과 `···`만 표시하고 모드 전환은 `···` 메뉴에 둔다.
- 키보드 포커스가 Daymark에 있을 때 `Ctrl+Shift+D`로 전환한다.
- 복구가 필요하면 `python -m daymark.main --window`를 사용한다.
- 연결 실패를 오류 모달 반복으로 방해하지 않고 일반 창으로 폴백한다.
- 설정에서 모니터를 바꾸면 현재 바탕화면 모드 위치를 즉시 갱신한다.
- 일반 창에서 바탕화면 모드로 전환할 때는 창이 현재 위치한 모니터를 우선한다.
- 저장된 모니터 번호가 유효하지 않으면 주 모니터로 폴백한다.

## Transparency

- 기본 불투명도 `0.94`
- 설정에서 `0.78`~`1.0` 조절, 즉시 미리보기 및 저장
- Windows에서는 불투명 전경과 반투명 배경을 분리해 글자가 흐려지지 않게 한다.
- `DAYMARK_OPACITY` 환경변수는 실행 시 설정값을 임시로 덮어씀
- 창/바탕화면 모드 전환 후 Tk alpha와 Win32 layered alpha를 재적용
- 미지원 환경은 불투명 폴백
- 반투명 효과는 데이터 저장과 독립


## Toolbar and More Menu

- 상단 중앙에는 이전 달, 연월, 다음 달, 오늘만 둔다.
- 상단 우측에는 `AI 요약`과 `···`만 둔다.
- `···`는 모달이 아닌 작은 평면 팝업이며 바깥 클릭과 `Esc`로 닫힌다.
- 미완료 오늘로 이동, 설정, 모드 전환, 앱 종료는 팝업 안에 둔다.
- 앱 종료만 약한 위험 색상을 사용할 수 있다.

## Typography and Color Tokens

- `Segoe UI Variable` → `Segoe UI` → `TkDefaultFont` 순으로 폴백한다.
- 월 18pt, 요일 10pt, 날짜 11pt, 업무 11pt를 공통 토큰으로 사용한다.
- 선택 배경은 `#272A2E`, 토요일은 `#7893B8`, 일요일·공휴일은 `#C97C82`를 기준으로 한다.
- 완료 업무는 녹색 없이 원형 체크와 `MUTED_TEXT` 명암으로만 구분한다.

## Keyboard Rules

| 상황 | 키 | 결과 |
|---|---|---|
| 날짜 셀 클릭 | 마우스 | 날짜 선택 및 빈 행 포커스 |
| 빈 행 입력 | `Enter` | 저장 후 다음 빈 행 생성 |
| 기존 업무 수정 | `Enter` | 수정 저장 후 다음 행 포커스 |
| 기존 업무가 빈 상태 | `Backspace` | 업무 삭제 |
| 기존 업무 수정 후 이동 | 포커스 아웃 | 수정 저장 |
| Daymark 포커스 상태 | `Ctrl+Shift+D` | 창/바탕화면 모드 전환 |

## Visual States

- 오늘: 파란 원형 배경과 밝은 날짜 숫자
- 토요일: 낮은 채도의 파란 날짜, 더 흐린 파란 요일 헤더
- 일요일 및 대한민국 공휴일: 낮은 채도의 빨간 날짜, 더 흐린 빨간 요일 헤더
- 선택 날짜: 카드처럼 보이지 않는 낮은 대비의 셀 배경
- 현재 월 밖 날짜: 날짜와 업무를 주말색과 배경색 사이의 흐린 명암으로 표시
- 미완료: 얇은 원형 테두리와 기본 글자
- 완료: 원형 체크 표시와 흐린 글자
- 빈 입력 행: 평소 배경과 동일, 포커스에서만 약한 변화
- 날짜 셀: 항상 무테두리·무그림자

## Incomplete Copy

- `미완료 오늘로 이동`은 오늘 이전의 모든 미완료 업무를 오늘에 추가한다.
- 과거 날짜의 원본은 그대로 보인다.
- 같은 날 반복 클릭은 중복을 만들지 않는다.
- 복사 결과는 포커스를 빼앗지 않는 약 3초짜리 하단 중앙 토스트로 알린다.

## Dialogs

보고서와 설정 대화상자는 메인과 같은 어두운 평면 스타일을 사용한다. 바탕화면 모드에서 연 대화상자는 최초 표시 순간만 앞으로 올려 사용자가 찾을 수 있게 한다.

## Error UX

- WorkerW 없음: 일반 창으로 폴백
- Explorer 재시작: 조용히 재연결 시도
- API 키 없음: 요청을 보내지 않고 설정 방법 안내
- LLM 오류: 업무 입력은 계속 가능
- DB 오류: 앱 시작을 중단하고 원인을 숨기지 않음

## Overflow

- 업무가 날짜 셀 높이를 넘을 때만 얇은 세로 스크롤을 표시한다.
- `Enter`로 생성한 다음 입력 행은 자동으로 보이는 위치까지 스크롤한다.
- 날짜 셀은 계속 무테두리·무그림자를 유지한다.
