# UX Specification

## Visual Direction

Daymark는 카드형 일정 앱이 아니라 바탕화면 위의 텍스트 달력처럼 보여야 한다.

- 월 이동은 가로 중앙
- 날짜 셀은 무테두리·무흰칸
- 저장된 업무만 `□ / ✓` 표시
- 토요일은 파란색
- 일요일과 대한민국 공휴일은 빨간색

## Incomplete Copy

- `미완료 복사`는 오늘 이전 모든 미완료 업무를 오늘에 추가한다.
- 과거 원본은 그대로 남는다.
- 같은 날 반복 클릭은 중복을 만들지 않는다.
- 최신 복사본을 완료하면 다음 날 다시 복사하지 않는다.
- 완료 메시지는 복사 건수와 원본 유지 사실을 안내한다.

## Desktop Mode

- 일반 창에서 바탕화면 모드로 전환하면 현재 창이 위치한 모니터를 선택한다.
- 설정에서 명시적으로 모니터를 바꿀 수 있다.
- 바탕화면 모드는 선택 모니터 한 대만 채운다.
- `Ctrl+Shift+D`로 모드를 전환한다.
- 전환 후에도 설정된 투명도를 유지한다.
- WorkerW 연결 실패 시 일반 창으로 복구한다.

## Keyboard

| 상황 | 키 | 결과 |
|---|---|---|
| 빈 행 입력 | `Enter` | 저장 후 다음 행 |
| 기존 업무 수정 | `Enter` | 저장 후 다음 행 |
| 수정 후 이동 | 포커스 아웃 | 수정 저장 |
| Daymark 포커스 | `Ctrl+Shift+D` | 창/바탕화면 전환 |
